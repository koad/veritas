# Fact-Check Review: Day 11 — "Files on Disk Beats Cloud"

**Verdict: NEEDS_REVISION**

One factual error requires correction before publication. All five cited incidents check out against the brief at the claim level. Technical claims are accurate. Steelman is fairly stated. Two minor omissions noted but not blocking.

## Critical Error

### "You have thirteen months" (paragraph 1, last sentence of opening section)

**Draft:** "The Assistants API would be shut down on August 26, 2026 ... You have thirteen months."

**Brief:** "August 2025: Assistants API deprecation announced; sunset date August 26, 2026."

**Finding:** INCORRECT. August 2025 to August 26, 2026 is twelve months, not thirteen. The brief does not specify the exact date within August 2025 when the deprecation was announced, but even if the announcement arrived on August 1, 2025, the lead time to August 26, 2026 is twelve months and twenty-five days — rounded correctly as "twelve months," not thirteen. "Thirteen months" overstates the notice period by a full month, which weakens the precision of the opening hook and could be corrected by anyone who checks the dates.

**Correction:** Change "thirteen months" to "twelve months."

## Technical Claims

### Git as content-addressable storage

The draft does not explicitly claim "content-addressable storage" — it describes git's behavioral properties (audit trail, branching, rewinding, cloning). No false technical claim is made by omitting the internal implementation term. PASS.

### GPG-signed commits

The draft uses "cryptographically attributed" rather than "GPG-signed commits." This is accurate: koad:io's trust bond protocol uses GPG clearsigning, and the phrase "cryptographically attributed" correctly describes what git commits with signing provide. The draft does not overclaim. PASS.

### Branching, forking, rewinding

Draft: "An agent that lives in a git repository can be cloned, diffed, and branched. Revert to yesterday's identity if a session went badly. Run parallel experiments. Audit the exact state at any commit."

All four claims are technically accurate descriptions of standard git capabilities. PASS.

### "Files on disk" audit trail via `git log`

Draft uses `git log --oneline` as the audit trail. This is accurate: committed files in a git repo produce a human-readable audit trail via git log. PASS.
