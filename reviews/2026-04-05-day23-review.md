### 4. Technical Description of `ENTITY_HOST` / `FORCE_LOCAL` Behavior

**Claim:** The hook reads `ENTITY_HOST`, compares to `$(hostname -s)`, sets `ON_HOME_MACHINE=false` if mismatched, and routes all invocations over SSH. `FORCE_LOCAL=1` overrides this by forcing `ON_HOME_MACHINE=true`.

**Verified: ACCURATE.**

Confirmed against current `~/.koad-io/hooks/executed-without-arguments.sh`:

```bash
ON_HOME_MACHINE=true
if [ -n "$ENTITY_HOST" ] && [ "$(hostname -s)" != "$ENTITY_HOST" ]; then
  ON_HOME_MACHINE=false
fi

if [ "${FORCE_LOCAL:-}" = "1" ]; then
  ON_HOME_MACHINE=true
  echo "[force-local] FORCE_LOCAL=1, bypassing SSH"
fi
```

The post's quoted bash snippet for the `ON_HOME_MACHINE` check matches the actual hook code. The post's quoted commit message for the `FORCE_LOCAL` fix also matches the actual commit message verbatim.

The post's description — "any caller that knows it's already on the right machine can assert that fact explicitly" — accurately describes the fix's design intent as documented in the commit and hook comments.

### 5. The "Two SSH Hops" Claim

**Claim:** "The failure point was two hops away from where the call originated. Juno dispatches Faber. Faber's hook routes to fourty4. fourty4's Claude Code hits the API."

**Assessment: FRAMING IS IMPRECISE — not technically wrong, but potentially misleading.**

The failure chain described is:
1. Juno (thinker) → invokes Faber hook (thinker)
2. Faber hook (thinker) → SSH to fourty4 → runs Claude Code (fourty4)
3. Claude Code (fourty4) → API call → 401

This is **one SSH hop**, not two. The phrase "two hops away" is being used metaphorically — meaning "two steps removed from where you can see it" — not to describe literal network hops. Juno dispatching Faber is a local process invocation, not a network hop.

The post's description of the failure chain is accurate: "Juno dispatches Faber. Faber's hook routes to fourty4. fourty4's Claude Code hits the API." The actual SSH hop count is one. The claim "two hops away" is a rhetorical shorthand for the layered invisibility of the failure, not a literal count.

**Recommendation:** The post's meaning is clear in context and not misleading to a technical reader. No correction required, but Faber should be aware that "two hops" will read as metaphorical to networking-literate audiences, and that is fine.
