## 5. Hook Implementation — Code Accuracy and Line Numbers

**Claim:** "Here is the implementation. This is `~/.koad-io/hooks/executed-without-arguments.sh`, lines 34–38 — the entirety of the pre-invocation assembly mechanism."

The code block cited in the post:

```bash
if [ -n "$PROMPT" ] && [ -f "${CALL_DIR}/PRIMER.md" ]; then
  PROJECT_PRIMER="$(cat "$CALL_DIR/PRIMER.md")"
  PROMPT="$(printf 'Project context (from %s/PRIMER.md):\n%s\n\n---\n\n%s' \
    "$CALL_DIR" "$PROJECT_PRIMER" "$PROMPT")"
fi
```

**Verdict: Functionally accurate. Minor cosmetic presentation difference. Non-blocking.**

The actual file at `~/.koad-io/hooks/executed-without-arguments.sh` lines 34–38 reads:

```bash
if [ -n "$PROMPT" ] && [ -f "${CALL_DIR}/PRIMER.md" ]; then
  PROJECT_PRIMER="$(cat "$CALL_DIR/PRIMER.md")"
  PROMPT="$(printf 'Project context (from %s/PRIMER.md):\n%s\n\n---\n\n%s' "$CALL_DIR" "$PROJECT_PRIMER" "$PROMPT")"
  echo "[primer] Injected PRIMER.md from $CALL_DIR ($(wc -c < "$CALL_DIR/PRIMER.md") bytes)"
fi
```

Two differences:

1. **Line 37 is an `echo` debug line** — `echo "[primer] Injected PRIMER.md from $CALL_DIR ($(wc -c < "$CALL_DIR/PRIMER.md") bytes)"` — that the post omits. This is a debug/logging statement, not part of the assembly logic. Its omission from the "mechanism" excerpt is editorially accurate: the post is showing the assembly mechanism, not the full function. The omission does not misrepresent what the code does.

2. **The `printf` line is split across two lines in the post** for readability; in the actual file it is a single line. The content is identical. This is a formatting choice, not a code change.

**The line count claim of "five lines" is accurate for the logical mechanism** (the if/fi block minus the debug echo is four non-blank lines; the post counts five including the closing `fi`). The mechanism description is accurate.

**Note for Faber:** If the post is intended as a literal code excerpt, adding the omitted `echo` line would make it a verbatim reproduction. Given the post's framing ("the entirety of the pre-invocation assembly mechanism"), omitting a debug log line is defensible. No change required; optional improvement only.

## 6. The `$CWD` Variable Name in the Routing Table

**Claim (table heading and body):** `$CWD/PRIMER.md` appears in the comparison table as the AI Entity equivalent of `$HOME/.bashrc`.

**Verdict: Minor naming imprecision. Non-blocking, but flag for awareness.**

The hook resolves working directory as `CALL_DIR="${CWD:-$PWD}"` — it uses a custom `$CWD` variable (or falls back to `$PWD`) rather than the standard POSIX `$PWD`. The hook's variable name is `$CWD`, exported from the calling context, not a standard shell variable. Using `$CWD/PRIMER.md` in the table is therefore technically consistent with the hook's internal variable naming, but could confuse readers who expect `$CWD` to behave like the POSIX `$PWD`.

The routing discussion in the post uses "working directory" as the plain-English concept throughout, which is accurate. The `$CWD` appearance is confined to the table. No misrepresentation of the mechanism — the routing is genuinely working-directory-based. Flag only, not a required correction.
